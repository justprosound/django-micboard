QUICK START FOR NEW CONTRIBUTORS
================================================================================

1. CLONE REPOSITORY:
   git clone https://github.com/username/django-micboard.git
   cd django-micboard

2. SETUP ENVIRONMENT:
   cp .env.example .env.local
   # Edit .env.local with your local values

3. INSTALL & RUN:
   # See docs/CONFIGURATION.md for detailed setup
   source .env.local
   python manage.py migrate
   python manage.py runserver

4. TEST LOCAL API:
   docker compose -f demo/docker/docker-compose.yml up -d
   # API will be at localhost:8000

5. (OPTIONAL) CONNECT VPN DEVICES:
   # Edit .env.local with your device IPs
   SHURE_DEVICE_IPS=172.21.1.100,172.21.1.101,...
   
   # Run discovery
   python scripts/device_discovery.py discover --env
   
   # See docs/VPN_DEVICE_POPULATION.md for details

WHAT'S PROTECTED FROM PUBLIC VIEW
================================================================================

These files/information are automatically excluded via .gitignore:
  ✓ .env.local                    - Your personal environment config
  ✓ device_manifest.json          - Local device discovery results  
  ✓ *.key, *.pem                  - SSL certificates
  ✓ sharedkey.txt                 - Shure API authentication
  ✓ db.sqlite3                    - Local database
  ✓ scripts/local_*.py            - Personal test scripts

================================================================================
IMPORTANT RULES
================================================================================

❌ NEVER COMMIT:
  - .env.local or any .env file with real values
  - Hardcoded API keys, passwords, or shared keys
  - VPN device IP addresses in source code
  - Database files or local test data
  - Screenshots containing sensitive information

✅ ALWAYS:
  - Use environment variables for configuration
  - Use .env.local for personal configuration
  - Review changes before committing (git diff)
  - Check for secrets in git log (git log --all -p | grep -i secret)
  - Read SECURITY.md before your first commit
